# 1단계 분류기 연동 규약 (팀원 ↔ 백엔드)

> 1단계 분류기는 **팀원이 `klue/roberta-base`로 별도 학습**한다.
> 백엔드가 그 모델을 그대로 불러오려면 아래 규약만 지키면 된다.

## 1. 라벨 규약 (가장 중요)
3클래스, **id 순서 고정**:

| id | label |
|----|-------|
| 0 | 정상 |
| 1 | 주의 |
| 2 | 위험 |

`AutoModelForSequenceClassification` 학습 시 `id2label`/`label2id`를 위 순서로 설정.
(백엔드 `ml/classifier/labels.py` 와 동일해야 함)

## 2. 저장 형식
HuggingFace 표준으로 **모델 + 토크나이저를 같은 폴더에** 저장:
```python
model.save_pretrained("klue-roberta-clf")
tokenizer.save_pretrained("klue-roberta-clf")
```
→ `config.json`, `model.safetensors`, `tokenizer.json` 등이 한 폴더에.

## 3. 백엔드 연동
저장 폴더 경로를 환경변수로 지정하면 끝:
```bash
CLASSIFIER_MODEL_DIR=/path/to/klue-roberta-clf uvicorn app.main:app --reload
```
- 미지정 시 → 휴리스틱 폴백(데모용)
- 지정 시 → 백엔드 `nlp/pipeline.py` 가 해당 모델로 분류
- 어느 경우든 **긴급 룰**(학폭·아동학대·자해·고소)은 위험으로 강제 승격(백엔드가 처리)

## 4. 학습 데이터
- 변환 스크립트 재사용 가능: `ml/classifier/prepare_aihub.py` → `data/processed/complaints_{train,val}.csv` (text,label)
- 팀원이 자체 코드로 학습해도 되고, `ml/classifier/`(백본만 `CLF_MODEL=klue/roberta-base`)를 써도 됨:
  ```bash
  cd ml
  CLF_MODEL=klue/roberta-base python -m classifier.train --data ../data/processed/complaints_train.csv
  ```

## 5. 모델 파일 공유
- 모델 가중치(`*.safetensors`)는 `.gitignore` 대상(대용량) → git 말고 별도 공유(드라이브 등)
- 또는 합의 시 작은 모델은 git-lfs 검토
